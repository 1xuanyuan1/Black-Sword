# Project instructions

## 项目文档路由

项目的长期设计依据位于 [`doc/`](doc/README.md)。开始任务前只读取与任务直接相关的文档：

- 玩法、平衡、奖励、角色能力、技能、心法、进阶、道具、波次、关卡或美术需求，读取 [`doc/v1_game_design.md`](doc/v1_game_design.md)。
- 世界观、三国时代背景、人物关系、剧情触发、对白、命名或界面文字，读取 [`doc/v1_narrative_design.md`](doc/v1_narrative_design.md)。
- Godot 场景与脚本架构、Resource 数据、存档、性能、自动测试或实施顺序，读取 [`doc/v1_technical_design.md`](doc/v1_technical_design.md)。
- 跨领域任务组合读取涉及的文档，不需要无差别读取全部资料。例如实现小 Boss 宝箱时读取策划案的宝箱规则与技术方案的进阶接口。

策划案是玩家可观察行为与数值的来源，文案设计是世界观与最终文字的来源，技术方案是代码接口与持久化格式的来源。文档发生冲突时，不要在代码中自行选择；先更新对应文档并同步相关引用，再实施改动。

修改公开内容 ID、存档字段或解锁条件时，必须同步更新技术方案中的迁移与测试要求。修改外部素材时，还必须遵守下方授权与来源记录规则。

## 素材获取与制作

- 处理角色、怪物、Boss、技能特效、地图、瓦片、动画帧或 Sprite Sheet 等游戏素材任务时，优先检查 [FrameRonin](https://frameronin.com/) 是否已有合适素材或对应制作工具。
- 可优先使用 FrameRonin 的角色、怪物和场景生成，以及去背、像素化、切帧、Sprite Sheet 拆分/合成/调整、动画预览、地图拼接和测试场景等工具。
- 当前项目是 Godot 4.7 项目，不是 RPG Maker 项目。不要直接使用 RPG Maker 专用图集布局；确需使用时，必须先转换为适合 Godot 的帧布局并完成导入验证。
- 下载或生成素材后，必须检查透明背景、像素尺寸、单帧宽高、方向顺序、动作命名、缩放过滤方式和 Godot 4.7 导入兼容性。
- 新素材必须尽量匹配项目现有的暗色像素武侠风格；必要时先完成调色、缩放、去背和边缘清理，再导入游戏。
- 角色、怪物和 Boss 素材应整理为独立 `.tscn` 场景预设，节点名称应反映实际功能与职责。
- 不要把完整素材包无筛选地导入项目，只保留当前实际使用的文件，并放入职责清晰的资源目录。

## 授权与来源记录

- 导入任何外部素材前，必须确认许可证、商用范围、修改权限和再分发权限。
- 在 `CREDITS.md` 中记录素材名称、来源链接、作者或平台、许可证以及本项目所做的修改。
- 来源或授权不明确的素材只能用于内部原型，并须在 `CREDITS.md` 中明确标记；在确认授权前不得用于正式发布。
- FrameRonin 是素材查找和制作的优先入口，不代表网站内所有素材都自动获得可商用授权，每项素材仍需单独核对。
