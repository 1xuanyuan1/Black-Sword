## 阅读规则

- 调整玩法、平衡、奖励、角色能力、技能、波次、关卡或美术需求时，阅读 `v1_game_design.md`。
- 调整世界观、角色关系、剧情触发、对白、名称或界面文字时，阅读 `v1_narrative_design.md`。
- 调整 Godot 场景、脚本架构、Resource 数据、存档、性能或自动测试时，阅读 `v1_technical_design.md`。
- 跨领域任务只组合读取涉及的文档。例如“实现小 Boss 宝箱”应同时读取策划方案中的宝箱规则和技术方案中的进阶接口。
- 策划案是玩家可观察行为与数值的来源；文案设计是世界观和最终文字的来源；技术方案是代码接口与持久化格式的来源。
- 三份文档发生矛盾时，不直接在代码中自行选择。先修正文档并记录变更，再开始实现。

## 变更规范

修改文档时应同时完成以下事项：

1. 说明变更影响的系统和验收项。
2. 同步更新其他文档中引用同一概念的内容。
3. 若修改公开 ID、存档字段或解锁条件，在技术方案的兼容性章节补充迁移要求。
4. 若新增或替换外部素材，在项目根目录 `CREDITS.md` 记录来源、作者、许可证、修改内容和允许发布的平台。
5. 文档统一保存为 UTF-8 Markdown，项目内路径使用 Godot 的 `res://` 或仓库相对路径。

## V1 边界

V1 面向 Windows 与桌面 Web，移动端保留可玩能力。V1 不包含云存档、联机、付费系统、战斗中途续局、第二张地图和多语言本地化。以上内容若进入后续版本，应新建设计版本或明确修订本目录中的 V1 文档。
