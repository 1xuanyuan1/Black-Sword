# 黑剑·荒寺夜行

一款使用 Godot 4 制作的 2D 俯视角武侠肉鸽试玩版。玩家持黑剑夜闯荒寺，在不断增强的妖物围攻中修炼招式与心法，最终迎战 Boss「鬼面剑豪」。

![夜间荒寺地图](assets/world/arena_background.png)

## 当前内容

- 一张可探索的夜间荒寺地图，包含薄雾、月光和树干实体碰撞。
- 两名可选玩家角色：黑剑客与联动角色波风水门，分别以黑剑·横扫和螺旋丸开局。
- 四类普通敌人：尸傀、影犬、灯笼鬼、重甲怨卒。
- 四波逐步增强的敌群；第一波尸傀生命较低，3 分钟后怪物生命、刷新速度和场上密度进入高压阶段。
- Boss「鬼面剑豪」，包含近战斩击、冲锋、弹幕、环形剑阵和召唤小怪。
- 九个主动招式、两个被动心法，全部可升至 5 级。
- 标题、角色选择、数字生命值 HUD、暂停、升级三选一、胜负结算和重新开始流程。
- Master、Music、SFX 三条音频总线。

## 操作

| 操作 | 按键 |
| --- | --- |
| 移动 | `WASD` 或方向键 |
| 暂停 / 继续 | `Esc` |
| 选择角色 | 鼠标点击或数字键 `1`、`2` |
| 选择升级 | 鼠标点击或数字键 `1`、`2`、`3` |
| 攻击 | 自动索敌并释放 |

移动端与手机浏览器使用横屏布局：竖屏打开时会先显示旋转提示，不再拉长战斗画面；手指在左半屏任意位置按住即可唤出浮动摇杆，拖动控制、松手隐藏；右上角触摸按钮暂停，角色卡、技能三选一以及菜单按钮均可直接触摸选择。

升级界面出现时战斗会完全暂停。每局最多持有 4 个主动招式和 2 个被动心法。

## 技能

所有招式都会自动索敌和释放，单项最高 5 级。每局最多装备 4 个主动招式和 2 个被动心法；已经满级的技能以及没有空余对应槽位的新技能不会进入升级三选一。

黑剑客以「黑剑·横扫」开局，波风水门以「螺旋丸」开局。目前两个初始技能也都在公共技能池中，满足主动槽位条件时，另一名角色仍有机会在升级中习得它们。

伤害数值均为淬锋心法加成前的基础值，冷却也为冷却缩减前的基础秒数。范围、半径、速度和击退为 Godot 世界单位。

### 主动招式

#### 黑剑·横扫（`black_slash`）

黑剑客的初始招式。自动朝最近敌人斩出近身扇形剑弧，同时击退范围内的敌人，适合处理贴身怪群。5 级会在 0.18 秒后追加第二道剑弧。

| 等级 | 伤害 | 范围 | 剑弧（弧度） | 冷却 | 等级变化 |
| --- | ---: | ---: | ---: | ---: | --- |
| 1 | 18 | 92 | 1.80 | 0.90 | 基础剑式 |
| 2 | 25 | 102 | 1.95 | 0.82 | 剑弧更宽，伤害提升 |
| 3 | 33 | 112 | 2.10 | 0.74 | 进一步提高范围、伤害和出剑速度 |
| 4 | 44 | 124 | 2.25 | 0.66 | 剑势大幅增强 |
| 5 | 54 | 132 | 2.40 | 0.58 | 化境：连续释放双重剑弧 |

#### 螺旋丸（`rasengan`）

波风水门的初始招式。向最近敌人发射一枚会持续修正方向的追踪螺旋丸，并带有击退。2 级开始命中后产生范围冲击；范围伤害只结算给主目标以外的附近敌人。4 级开始爆炸后分裂出更快的追踪小螺旋丸，分裂弹不会继续分裂。

| 等级 | 主体伤害 | 主体半径 | 速度 | 爆炸半径 / 伤害 | 分裂 | 冷却 | 等级变化 |
| --- | ---: | ---: | ---: | --- | --- | ---: | --- |
| 1 | 25 | 18 | 290 | 无 | 无 | 1.45 | 单体追踪螺旋丸 |
| 2 | 33 | 20 | 310 | 78 / 主体伤害的 65% | 无 | 1.32 | 解锁群体螺旋冲击 |
| 3 | 43 | 27 | 335 | 105 / 72% | 无 | 1.20 | 进化为更大的螺旋丸 |
| 4 | 55 | 29 | 360 | 116 / 78% | 3 枚，每枚 55% 伤害 | 1.08 | 命中后分裂追踪 |
| 5 | 72 | 38 | 390 | 150 / 85% | 4 枚，每枚 65% 伤害 | 0.95 | 化境：大玉螺旋丸与四重分裂 |

#### 飞剑诀（`flying_sword`）

向最近敌人发射自动追踪飞剑。等级提高后增加飞剑数量、贯穿、速度和树干反弹次数；5 级飞剑飞行至约一半寿命时折返，并能再次命中沿途目标。

| 3 | 26 | 2 | 440 | 2 | 1 | 1.72 | 贯穿能力提升 |

朝最近敌人释放宽幅直线剑气，能一次贯穿多名敌人。3 级开始可以被树干反弹；5 级会额外释放一道与主剑气垂直的交叉剑气。

### 被动心法

#### 轻身诀（`light_step`）

直接提高玩家移动速度和经验球拾取范围。加成为累计总值，不会在上一等级的结果上重复相乘；基础拾取范围为 92。

| 等级 | 移动速度 | 拾取范围 | 实际拾取范围 |
| --- | ---: | ---: | ---: |
| 1 | +6% | +12% | 103.04 |
| 2 | +12% | +24% | 114.08 |
| 3 | +18% | +36% | 125.12 |
| 4 | +24% | +48% | 136.16 |
| 5 | +30% | +60% | 147.20 |

#### 淬锋心法（`tempered_edge`）

提高所有玩家技能的最终伤害，并按比例缩短主动招式的基础冷却。加成为累计总值；例如 5 级时，54 点基础伤害会变为 75.6，1 秒基础冷却会变为 0.8 秒。

| 等级 | 全局伤害 | 冷却缩减 |
| --- | ---: | ---: |
| 1 | +8% | 4% |
| 2 | +16% | 8% |
| 3 | +24% | 12% |
| 4 | +32% | 16% |
| 5 | +40% | 20% |

### 技能代码位置

当前技能不是十一个各自独立的 `.tscn` 预设，而是由战斗场景中的 `GameplaySystems/SkillSystem` 节点统一管理，并在运行时创建弹丸和特效。

| 文件 | 当前职责 |
| --- | --- |
| [`scripts/content_registry.gd`](scripts/content_registry.gd) | 所有技能 ID、中文名、说明、颜色和 1～5 级数值；技能平衡主要修改这里 |
| [`scripts/data/skill_definition.gd`](scripts/data/skill_definition.gd) | `SkillDefinition` 数据结构、最大等级、当前等级数值读取和升级文案 |
| [`scripts/skill_system.gd`](scripts/skill_system.gd) | 主动槽/心法槽、升级候选、冷却、自动索敌、施法行为和被动加成 |
| [`scripts/combat_projectile.gd`](scripts/combat_projectile.gd) | 螺旋丸、飞剑和剑气的移动、追踪、贯穿、折返、碰撞反弹、爆炸与分裂 |
| [`scripts/effect_node.gd`](scripts/effect_node.gd) | 剑弧、落雷、霜阵、冲击波、剑雨等运行时特效绘制 |
| [`scripts/data/damage_event.gd`](scripts/data/damage_event.gd) | 技能传递给目标的伤害、方向、击退和伤害标签 |
| [`scripts/enemy_actor.gd`](scripts/enemy_actor.gd) | 普通敌人受伤、击退、减速和冻结的实际处理 |
| [`scripts/arena.gd`](scripts/arena.gd) | 将角色初始技能交给技能系统，触发升级三选一并接收升级结果 |
| [`scripts/main.gd`](scripts/main.gd) | HUD 技能槽与升级选择界面的显示、按键和鼠标选择 |
| [`scenes/gameplay/battle_arena.tscn`](scenes/gameplay/battle_arena.tscn) | 挂载 `SkillSystem` 节点的完整战斗场景 |
| [`scenes/actors/player.tscn`](scenes/actors/player.tscn) | 黑剑客预设，`initial_skill_id = black_slash` |
| [`scenes/actors/player_minato.tscn`](scenes/actors/player_minato.tscn) | 波风水门预设，`initial_skill_id = rasengan` |

回风护体目前使用 [`assets/vfx/dragon_tornado.png`](assets/vfx/dragon_tornado.png) 作为六帧龙卷动画；螺旋丸、飞剑、剑气以及大部分范围特效目前由 GDScript 动态绘制。

## 运行项目

推荐使用 **Godot 4.7 stable**。

1. 使用 Godot Project Manager 导入根目录下的 `project.godot`。
2. 打开项目后按 `F6` 可单独预览当前场景，按 `F5` 从标题页启动完整游戏。
3. 首次打开时等待 Godot 完成资源导入。

也可以从命令行启动：

```bash
godot --path .
```

macOS 默认安装位置下可使用：

```bash
/Applications/Godot.app/Contents/MacOS/Godot --path .
```

### Web 导出与测试地址

项目包含无多线程 Web 导出预设，可在普通静态 HTTP 服务器上运行：

```bash
godot --headless --path . --export-release Web builds/web/index.html
```

当前测试站点为 [https://game.dduke.cn](https://game.dduke.cn)，HTTP 请求会自动跳转到 HTTPS。Nginx 配置保存在 [`deploy/nginx/game.dduke.cn.conf`](deploy/nginx/game.dduke.cn.conf)，Web 横屏启动页保存在 [`assets/web/game_web_shell.html`](assets/web/game_web_shell.html)。Web 构建产物位于 `builds/web/`，该目录已加入 `.gitignore`，不应提交到源码仓库。

## 自动验证

运行资源导入与脚本解析检查：

```bash
godot --headless --path . --editor --quit
```

运行完整自动测试：

```bash
godot --headless --path . --script tests/test_runner.gd
```

测试覆盖角色与地图预设结构、技能数量和等级上限、槽位过滤、升级暂停、碰撞与反弹、后期敌人血量、Boss 唯一生成及胜利结算。

## AI资源库插件

项目已安装并启用 `AI资源库` 编辑器插件。重新打开 Godot 编辑器后，右侧 Dock 会出现“AI资源库”面板，可用于扫描 NPC JSON、预览角色图集、编辑台词并把生成的 NPC 拖入 2D 场景。

资源放置约定：

```text
AI资源库/
├── 一图全动作/<角色目录>/npc.json
├── RPGMAKER/<角色目录>/NPC.json
├── _schema/                # JSON Schema
└── _templates/             # NPC 与台词模板
```

插件生成的角色场景默认保存在 `generated/npc_library_tool/npcs/`。详细使用方法见 [AI资源库/README.md](AI资源库/README.md)。

插件包原本附带的示例资源已按用户要求导入，分别显示在“一图全动作”“RPG Maker”“火影忍者”等分类。插件与联动角色素材均未附许可证，因此只能先用于本地原型验证；对外发布前需要确认插件作者及角色素材权利方授权，详见 [CREDITS.md](CREDITS.md)。

## 开发说明

- 战斗数据集中在 `scripts/content_registry.gd`，新增或平衡技能、敌人与波次时优先从这里修改。
- 角色节点统一按职责命名，例如 `CharacterVisual`、`CharacterSprite`、`BodyCollision` 和 `FollowCamera`。
- 普通敌人与 Boss 通过对应 `PackedScene` 实例化，不再由脚本临时创建角色节点。
- 墙体只保留视觉，不设置空气墙；当前仅树干参与世界碰撞。
- `assets/actors/hero/hero_actual.png` 是当前玩家图集；旧武士素材 `samurai_blue.png` 仍保留，便于后续制作其他角色。
- `AI资源库/火影忍者/波风水门/sprite.png` 同时供 AI 资源库插件和水门玩家预设使用。
- 首版玩法面向桌面键盘与鼠标。项目包含导出预设，但暂未适配触屏操作、联网、存档或局外成长。
